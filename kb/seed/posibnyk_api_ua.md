# Посібник з API

Acme Cloud Suite надає REST API для побудови інтеграцій, автоматизацій та власних інструментів
на основі даних вашого робочого простору.

## Базова адреса

Усі запити до API виконуються за адресою:

```
https://api.acmecloudsuite.example.com/v1
```

## Автентифікація

API Acme Cloud Suite використовує автентифікацію за допомогою API-ключа. Створіть ключ у розділі
**Налаштування > Розробникам > API-ключі**, а потім додавайте його до заголовка `Authorization`
кожного запиту як bearer-токен:

```
Authorization: Bearer <key>
```

Запити без коректного заголовка `Authorization`, а також із простроченим або відкликаним
ключем, будуть відхилені.

### Приклад запиту

```bash
curl -X GET "https://api.acmecloudsuite.example.com/v1/tasks" \
  -H "Authorization: Bearer <key>" \
  -H "Content-Type: application/json"
```

Успішна відповідь повертає тіло у форматі JSON зі статусом `200 OK`.

## Ліміти запитів

Ліміти запитів до API залежать від тарифу вашого акаунта:

| Тариф | Ліміт запитів |
|---|---|
| Starter | 100 запитів/хвилину |
| Pro | 600 запитів/хвилину |
| Enterprise | Індивідуальний ліміт (узгоджується з менеджером акаунта) |

Ліміти застосовуються окремо для кожного API-ключа, залежно від тарифу робочого простору. Якщо
вашій інтеграції потрібен вищий ліміт, ніж передбачає тариф, клієнти Enterprise можуть узгодити
індивідуальний ліміт зі своїм персональним менеджером акаунта.

## Поширені коди помилок

| Статус | Значення | Типова причина |
|---|---|---|
| `401 Unauthorized` | Запит не містить дійсного API-ключа, або ключ недійсний/прострочений/відкликаний. | Відсутній заголовок `Authorization`, некоректний bearer-токен, або ключ був змінений/відкликаний у налаштуваннях. |
| `429 Too Many Requests` | Перевищено ліміт запитів вашого тарифу. | Надсилання більшої кількості запитів за хвилину, ніж дозволяє тариф (100/хв на Starter, 600/хв на Pro). Зменшіть частоту запитів і повторіть спробу після часу, вказаного в заголовку відповіді `Retry-After`. |

Також можуть повертатися інші стандартні HTTP-коди (`400` для некоректних запитів, `403` для
проблем з правами доступу, `404` для відсутніх ресурсів, `500` для помилок сервера) — деталі
дивіться в полі `message` тіла відповіді.

## Рекомендації

- Зберігайте API-ключі в безпечному місці й ніколи не додавайте їх до системи контролю версій.
- Періодично оновлюйте API-ключі в розділі **Налаштування > Розробникам > API-ключі**.
- Реалізуйте логіку повторних спроб із затримкою для відповідей `429`, враховуючи заголовок
  `Retry-After`.
- Використовуйте окремий API-ключ для кожної інтеграції, щоб мати змогу відкликати доступ однієї
  інтеграції, не впливаючи на інші.

З питаннями щодо доступу до API на вашому тарифі або щодо індивідуального ліміту запитів на
тарифі Enterprise звертайтесь до підтримки згідно з SLA вашого тарифу: 48 годин для Free/Starter,
24 години для Pro або 4 години для Enterprise.
